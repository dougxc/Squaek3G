The Squawk Project

Squawk is a research virtual machine for the Java language that examines better ways of building virtual machines.  Most commercial virtual machines are written in low level languages such as C and assembler.  We believe that virtual machines can be written in higher level languages, such as those they execute themselves.

The main goal of the Squawk project is to write as much of the virtual machine as possible in Java, for portability, ease of debugging, and maintainability. Squawk was inspired by the Squeak project.  Squawk aims at a small footprint, it is Java compliant, and is CLDC 1.1-compatible.  Squawk is meant to be used in small, resource constraint devices.

At present, there are two different deployments of the Squawk VM to aim at different markets, namely, the very small footprint Java Card market and the wireless sensor and actuator device Sun(TM) SPOT:

  • Squawk for Sun SPOT (2004-present)
  • Squawk for Java Card 3.0 (2002-2003)

The overall architecture and design decisions of the Squawk project are discussed in the Squawk Virtual Machine page.

Other research uses of Squawk include: 

  • Squawk for Secure Devices, work by Andrew Crouch (intern winter 2004) under Bill Bush and Doug Simon's supervision, which describes the porting of the Secure KVM (SKVM) to Squawk.
  • Embedding Squawk into the Solaris 10 kernel, work by Hiroshi Yamauchi (intern summer 2004) under Mario Wolczko's supervision. Technical Report to appear, June 2006.
